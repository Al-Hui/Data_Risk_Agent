# Data Risk Agent v1

Демо-система на `Python + Streamlit` для идентификации риска данных по технологическим инцидентам.

Система анализирует входящую таблицу инцидентов, определяет возможные сценарии ухудшения качества данных, связывает их с бизнес-процессами и ИТ-услугами, сравнивает с уже зарегистрированными рисками и помогает пользователю принять решение: регистрировать новый риск, скорректировать формулировку, использовать существующий риск или дополнить его новым сценарием.

В демо-режиме вместо реального LLM используется детерминированный провайдер ответов из [data/demo_reasoning.json](data/demo_reasoning.json), чтобы можно было стабильно показать работу пайплайна и интерфейса.

## Что умеет

- пакетно анализирует таблицу инцидентов
- различает стандартные, нестандартные и нерелевантные кейсы
- извлекает идентификаторы контрактов и оферт
- определяет участие ИТ-услуг в бизнес-процессах
- формирует и валидирует сценарии риска данных
- агрегирует сценарии по сочетанию `бизнес-процесс + фронтальная ИТ-услуга`
- показывает уже имеющиеся риски на процессе и помогает выбрать правильное действие
- подбирает подходящие работы по инцидентам для переноса в мероприятия по риску данных
- ведет методологическую вкладку с метриками качества
- накапливает базу кейсов и обучающую обратную связь для последующей донастройки агента

## Основные экраны

### Вкладка `Риски для пользователей`

- фильтры и очередь кейсов
- рабочая карточка по выбранному сочетанию
- проект нового риска или проекта уточнения существующего риска
- редактирование описания риска и фронтальной ИТ-услуги
- действия с подтверждением `Да / Нет`
- блок имеющихся рисков
- сценарии риска
- подбор мероприятий

### Вкладка `Метрики качества`

- доля инцидентов с признаками риска данных
- доля отклоненных рисков
- доля скорректированных рисков
- динамика инцидентов с признаками риска данных
- динамика отклонений и корректировок
- база запросов и ответов по кейсам
- контур обучающей обратной связи

## Контур обучающей обратной связи

После пользовательских решений система сохраняет кейсы как обучающие примеры:

- что предложил агент
- что выбрал пользователь
- какое обоснование дал пользователь
- какой `reward` присвоен этому решению
- какой тип обучающего сигнала это означает

Поддерживаемые типы сигналов:

- `positive_acceptance`
- `supervised_correction`
- `routing_correction`
- `merge_decision`
- `scenario_boundary_decision`
- `negative_rejection`

Примеры сохраняются:

- в интерфейсе на вкладке `Метрики качества`
- в файл `data/feedback_cases.jsonl` по мере работы пользователей

Эта база предназначена для последующего обучения по предпочтениям, корректировкам и негативным сигналам. В текущей версии это не онлайн-обучение модели внутри приложения, а встроенный контур сбора качественной обратной связи.

```bash
python3 -m pytest
streamlit run streamlit_app.py
```

```bash
pytest -q
python3 - <<'PY'
import sys
sys.path.insert(0, 'src')
import streamlit_app
print('streamlit_app import ok')
PY
```

- `17 passed`
- `streamlit_app import ok`

## Технические заметки

- проект демонстрационный и использует подготовленные ответы вместо реального LLM
- регистрация риска и мероприятий изолирована через adapter layer
- Excel-выгрузка не является обязательной частью текущего релиза
- основной экспорт результатов реализован через `CSV`
